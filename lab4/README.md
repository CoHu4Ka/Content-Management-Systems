# USM Notes — WordPress Plugin

> Лабораторная работа №4 · Разработка плагина для WordPress

---

##  Описание

**USM Notes** — образовательный WordPress-плагин, добавляющий на сайт раздел «Заметки» с поддержкой приоритетов (High / Medium / Low) и даты напоминания.

### Возможности

- Кастомный тип записей (CPT) `usm_note`
- Кастомная таксономия «Приоритет» (`usm_priority`), иерархическая
- Метабокс с полем **Due Date** (дата напоминания) в редакторе записей
  - Обязательное поле
  - Нельзя выбрать прошедшую дату
  - Верификация nonce для безопасности
- Отображение «Due Date» отдельной колонкой в списке записей
- Шорткод `[usm_notes]` с фильтрами по приоритету и дате
- Адаптивные CSS-стили карточек

---

## Установка и запуск

### Требования

- WordPress 5.8+
- PHP 7.4+
- Локальная среда: [LocalWP](https://localwp.com/), XAMPP, MAMP или аналог

### Шаги

1. Скопируйте папку `usm-notes` в `wp-content/plugins/`.
2. В панели администратора перейдите в **Плагины → Установленные**.
3. Нажмите **Активировать** напротив **USM Notes**.
4. Убедитесь, что в `wp-config.php` включён режим отладки:
   ```php
   define('WP_DEBUG', true);
   ```
5. После активации перейдите в **Настройки → Постоянные ссылки** и нажмите **Сохранить** — это обновит правила `.htaccess`.

## Документация плагина

### Custom Post Type: `usm_note`

Зарегистрирован функцией `usm_notes_register_cpt()` на хуке `init`.

| Параметр | Значение |
|---|---|
| `public` | `true` |
| `has_archive` | `true` |
| `slug` | `usm-notes` |
| `supports` | title, editor, author, thumbnail |
| `menu_icon` | `dashicons-sticky` |

---

### Таксономия: `usm_priority`

Зарегистрирована функцией `usm_notes_register_taxonomy()` на хуке `init`.

Рекомендуемые термины, которые нужно создать вручную в **USM Notes → Priority**:

| Название | Slug |
|---|---|
| High | `high` |
| Medium | `medium` |
| Low | `low` |

---

### Метабокс «Due Date»

- Расположен на боковой панели редактора CPT.
- Сохраняет значение в `_usm_due_date` (формат `YYYY-MM-DD`).
- Валидация: поле обязательно, дата не может быть в прошлом.
- Безопасность: nonce `usm_notes_save_due_date`.
- Ошибки отображаются в метабоксе при повторном открытии записи (через `set_transient`).

---

### Шорткод `[usm_notes]`

#### Параметры

| Параметр | Тип | Описание |
|---|---|---|
| `priority` | string | Slug приоритета (`high`, `medium`, `low`) |
| `before_date` | string | Показывать записи с Due Date ≤ указанной даты (формат `YYYY-MM-DD`) |

Если параметры не указаны — отображаются все опубликованные заметки.

## Примеры использования

### Все заметки

```
[usm_notes]
```

### Только заметки с высоким приоритетом

```
[usm_notes priority="high"]
```

### Заметки со сроком до 30 апреля 2025

```
[usm_notes before_date="2025-04-30"]
```

### Комбинированный фильтр

```
[usm_notes priority="medium" before_date="2025-06-01"]
```

### Страница «Все заметки» (пример содержимого)

```
<h2>Все заметки</h2>
[usm_notes]

<h2>Срочные</h2>
[usm_notes priority="high"]

<h2>До конца апреля</h2>
[usm_notes before_date="2025-04-30"]
```

## Ответы на контрольные вопросы

### 1. Разница между кастомной таксономией и метаполем

**Таксономия** — это система классификации: термины хранятся отдельно в таблицах `wp_terms` и `wp_term_taxonomy`, записи можно группировать, фильтровать в стандартных запросах, строить архивные страницы и виджеты.

**Метаполе (метаданные)** — произвольное значение, привязанное к конкретной записи; хранится в `wp_postmeta`.

| | Таксономия | Метаполе |
|---|---|---|
| Хранение | отдельные таблицы | `wp_postmeta` |
| Повторное использование | да (один термин → много записей) | нет |
| Архивная страница | да | нет (нужен кастомный шаблон) |
| Подходит для | категорий, тегов, уровней приоритета | дат, чисел, URL, флагов |

**Пример:** Приоритет (High/Medium/Low) — таксономия, потому что это конечный набор значений, и удобно фильтровать записи по нему. Дата напоминания — метаполе, потому что это уникальное для каждой записи число, по которому нужны сравнения (`<=`, `>=`).

---

### 2. Зачем нужен nonce при сохранении метаполей

**Nonce (Number Used Once)** — одноразовой токен, который генерируется при рендеринге формы (`wp_nonce_field`) и проверяется при сохранении (`wp_verify_nonce`).

Nonce защищает от атаки **CSRF (Cross-Site Request Forgery)**: злоумышленник не может заставить браузер авторизованного пользователя отправить поддельный POST-запрос на сохранение данных, потому что не знает актуальный токен.

**Если не проверять nonce:**
- Любой сайт может отправить запрос от имени вошедшего администратора.
- Данные могут быть перезаписаны без ведома пользователя.
- Возможна инъекция произвольных метаданных.

---

### 3. Важнейшие параметры `register_post_type()` и `register_taxonomy()`

#### `register_post_type()`

| Параметр | Роль во фронтенде и UX |
|---|---|
| `public` | Делает CPT доступным на фронтенде (URL, архив, поиск). Без этого записи существуют только в админке. |
| `has_archive` | Создаёт страницу-архив (например, `/usm-notes/`), что позволяет строить листинги без шорткода. |
| `rewrite` | Задаёт ЧПУ-слаг для единственной записи и архива. Влияет на SEO и читабельность URL. |
| `supports` | Определяет, какие стандартные блоки редактора доступны (заголовок, текст, миниатюра). Без `thumbnail` кнопки изображения не будет. |
| `labels` | Все надписи в меню, кнопках и уведомлениях. Без правильных labels интерфейс выглядит как «Post/Posts». |

#### `register_taxonomy()`

| Параметр | Роль во фронтенде и UX |
|---|---|
| `hierarchical` | При `true` — поведение как у категорий (дерево, чекбоксы); при `false` — как теги (поле ввода). Для приоритетов `true` логичнее. |
| `public` | Разрешает архивные страницы `/priority/high/` и добавляет таксономию в REST API. |
| `show_admin_column` | Автоматически добавляет колонку с терминами в список записей, улучшая обзор в админке. |
