##  Инструкции по запуску

### Предварительные требования

- PHP ≥ 7.4
- MySQL / MariaDB
- WordPress ≥ 6.0, установленный локально (XAMPP, WAMP, Laragon, Local by Flywheel и др.)

### Установка

1. **Скопируйте папку темы** в директорию WordPress:
   ```
   wp-content/themes/usm-theme/
   ```

2. **Активируйте режим отладки** в `wp-config.php`:
   ```php
   define('WP_DEBUG', true);
   define('WP_DEBUG_LOG', true);
   ```

3. **Активируйте тему** в панели администратора:
   ```
   WordPress Admin → Внешний вид → Темы → USM Theme → Активировать
   ```

4. **Настройте меню навигации**:
   ```
   Внешний вид → Меню → создайте меню и назначьте его расположению «Главное меню»
   ```

5. **Добавьте виджеты** в боковую панель:
   ```
   Внешний вид → Виджеты → Главный сайдбар
   ```

### Структура файлов

```
usm-theme/
├── style.css          # Обязательный файл с метаданными + все стили CSS
├── index.php          # Главный шаблон (главная страница / блог)
├── header.php         # Общая шапка (логотип, навигация)
├── footer.php         # Общий подвал (виджеты, копирайт)
├── sidebar.php        # Боковая панель (область виджетов)
├── single.php         # Отдельная запись блога
├── page.php           # Статические страницы WordPress
├── archive.php        # Архивы (категории, метки, авторы, даты)
├── comments.php       # Система комментариев
├── functions.php      # Функции, хуки, регистрация стилей и скриптов
├── screenshot.png     # Превью темы (1200×900px)
└── js/
    └── navigation.js  # Скрипт адаптивного мобильного меню
```

## Описание лабораторной работы

### Цель

Цель лабораторной работы №3 — **изучить архитектуру тем WordPress** и создать полноценную тему с нуля, без сторонних фреймворков.

### Основные этапы

#### Шаг 1 — Обязательные файлы
Любая тема WordPress требует как минимум двух файлов:
- **`style.css`** — со специальным блоком метаданных в CSS-комментарии (Theme Name, Author, Version и др.)
- **`index.php`** — резервный шаблон (fallback) для всех типов контента

#### Шаг 2 — Общие компоненты шаблонов
Повторяющиеся фрагменты кода вынесены в отдельные файлы:
- `header.php` — содержит `<head>`, навигацию, логотип
- `footer.php` — виджет-зоны подвала, копирайт, `wp_footer()`
- `sidebar.php` — боковая панель с зарегистрированными областями виджетов

Подключение осуществляется через стандартные функции WordPress:
```php
get_header();    // подключает header.php
get_footer();    // подключает footer.php
get_sidebar();   // подключает sidebar.php
```

#### Шаг 3 — Цикл WordPress (The Loop)
Отображение записей осуществляется через центральный механизм WordPress:
```php
if ( have_posts() ) :
    while ( have_posts() ) : the_post();
        the_title();
        the_excerpt();
        the_permalink();
    endwhile;
endif;
```

На главной странице выводятся **последние 5 записей** через `WP_Query` с параметром `posts_per_page => 5`.

#### Шаг 4 — functions.php
Центральный файл функций регистрирует:
- Поддержку миниатюр, динамического заголовка, логотипа, HTML5
- Меню навигации (`register_nav_menus`)
- Области виджетов (`register_sidebar`)
- Стили и скрипты (`wp_enqueue_style`, `wp_enqueue_script`)

#### Шаг 5 — Специализированные шаблоны
Иерархия шаблонов WordPress определяет, какой файл используется:
- `single.php` → страница отдельной записи блога
- `page.php` → статические страницы WordPress
- `archive.php` → архивы по категориям, меткам, авторам, датам
- `comments.php` → подключается в `single.php` и `page.php`

#### Шаг 6 — Стилизация
Тема использует **CSS Custom Properties** (переменные) для единообразия оформления. Дизайн адаптивен — корректно отображается на мобильных устройствах, планшетах и десктопах.

| `--radius` | `8px` | Скругление углов |
| `--shadow` | `0 2px 16px rgba(...)` | Тень карточек |

### Используемые хуки

```php
add_action( 'after_setup_theme', 'usm_theme_setup' );    // настройка темы
add_action( 'wp_enqueue_scripts', 'usm_theme_scripts' ); // стили + скрипты
add_action( 'widgets_init', 'usm_theme_widgets_init' );  // области виджетов
add_filter( 'excerpt_length', 'usm_custom_excerpt_length', 999 ); // длина анонса
add_filter( 'body_class', 'usm_body_classes' );          // классы тега body
```

## Примеры использования

### Вывод последних 5 записей (index.php)

```php
$args = array(
    'posts_per_page' => 5,
    'post_status'    => 'publish',
);
$query = new WP_Query( $args );

while ( $query->have_posts() ) :
    $query->the_post();
    the_title( '<h2>', '</h2>' );
    the_excerpt();
endwhile;
wp_reset_postdata();
```

### Регистрация стиля темы (functions.php)

```php
function usm_theme_scripts() {
    wp_enqueue_style(
        'usm-theme-style',    // уникальный handle
        get_stylesheet_uri(), // URL к style.css темы
        array(),              // зависимости
        '1.0.0'               // версия (для сброса кэша)
    );
}
add_action( 'wp_enqueue_scripts', 'usm_theme_scripts' );
```

### Подключение шапки и подвала

```php
<?php get_header(); ?>

    <!-- содержимое страницы -->

<?php get_footer(); ?>
```

### Вспомогательная функция метаданных записи

```php
// В functions.php определена функция:
function usm_post_meta() {
    // выводит: дата · автор · категория
}

// Вызов в любом шаблоне:
usm_post_meta();
```

### Адаптивное мобильное меню (js/navigation.js)

Скрипт на чистом JavaScript (без jQuery) переключает класс `.toggled` на элементе `.main-navigation` при клике на кнопку-гамбургер. Поддерживает закрытие меню по клику вне его области и управление атрибутом `aria-expanded` для доступности.

##  Ответы на контрольные вопросы

### 1. Какие два файла обязательны для любой темы WordPress?

Любая тема WordPress требует **минимум двух файлов**:

**`style.css`** — файл стилей, который обязательно должен содержать блок метаданных в CSS-комментарии:
```css
/*
Theme Name: Название темы
Author: Автор
Version: 1.0.0
*/
```
WordPress читает эти метаданные, чтобы распознать тему и отобразить её в панели администратора.

**`index.php`** — главный шаблон, который служит резервным (fallback) для любого типа контента. Если, например, нет файла `single.php`, WordPress использует `index.php` для отображения отдельных записей.

---

### 2. Как подключаются общие части шаблонов (header, footer, sidebar)?

WordPress предоставляет специальные функции подключения, которые соблюдают иерархию шаблонов:

```php
get_header();    // подключает header.php (или header-{name}.php)
get_footer();    // подключает footer.php (или footer-{name}.php)
get_sidebar();   // подключает sidebar.php (или sidebar-{name}.php)
get_template_part( 'template-parts/content', 'single' ); // подключает частичный шаблон
```

Преимущество перед обычным `include`: эти функции запускают хуки WordPress (`get_header`, `get_footer` и др.), что позволяет плагинам и дочерним темам вмешиваться в процесс.

---

### 3. В чём разница между `index.php`, `single.php` и `page.php`?

Разница — в **типе контента**, который отображает каждый файл согласно иерархии шаблонов WordPress:

| Файл | Отображаемый контент | Когда используется |
|------|---------------------|-------------------|
| `index.php` | Универсальный fallback | Когда нет никакого другого подходящего шаблона. Обычно используется для списка записей блога. |
| `single.php` | **Отдельные записи** (тип `post`) | При переходе по URL конкретной записи блога. Отображает полный текст, комментарии, навигацию пред./след. |
| `page.php` | **Статические страницы** WordPress (тип `page`) | При переходе на страницы, созданные через «Страницы → Добавить». Обычно без даты/автора и навигации. |

Полная иерархия для отдельной записи: `single-{post-type}-{slug}.php` → `single-{post-type}.php` → `single.php` → `singular.php` → `index.php`

---

### 4. Какова роль файла `functions.php` в теме?

`functions.php` — **центральный файл конфигурации** темы, эквивалент плагина, специфичного для темы. Он:

- **Загружается автоматически** WordPress при каждом запросе, до загрузки любого шаблона
- **Расширяет функциональность** WordPress через хуки (`add_action`, `add_filter`)
- **Объявляет поддержку** функций WordPress: миниатюры, динамический заголовок, логотип, HTML5
- **Регистрирует стили и скрипты** через `wp_enqueue_style()` и `wp_enqueue_script()` — правильный способ подключения CSS/JS (в отличие от прямого включения в HTML)
- **Регистрирует области виджетов** (сайдбар, подвал) через `register_sidebar()`
- **Определяет вспомогательные функции**, доступные во всех шаблонах (например, `usm_post_meta()`)
- **Изменяет поведение WordPress** через фильтры (например, длина анонса записи)
